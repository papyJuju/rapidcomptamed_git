La comptabilité rapidcomptamed est destinée à une prise rapide des recettes. Elle permet aussi une intégration facile des dépenses, et un contrôle de ces dépenses. Enfin la comptabilité peut être triée, analysée, la gestion des immobilisations est possible, ainsi que l'édition du Grand Livre et la page recettes dépenses de la 2035.

Pour installer rapidcomptamed avec medintux: il faut placer le répertoire rapidcomptamed dans celui nommé "Programmes" de Medintux. Dans "bin" lancer "comptatriee", puis settings et setup. Ca installe des boutons dans DrTux. L'installation de la base de donnée de la comptabilité s'est installée sous SQLITE dans $HOME/.rapidcomptamed . On peut voir ce répertoire dans le dossier utilisateur sous Linux par ctrl+h. Si installe aussi le fichier config.ini. Si on veut installer la comptabilité sous MYSQL par exemple pour travail en réseau, il faut inscrire dans config.ini MYSQL à la place SQLITE.

Pour installer rapidcomptamed seul : lancer comptatriee, la base comptabilité sera installée avec SQLITE, et une base nommée "nameindex" de noms et prénoms sera créée.

Utiliser le module comptatriee. : ce module permet l'utilisation de toutes les ressources de la comptabilité. Commencez par Paramétrer la comptabilité : votre(vos) utilisateur(s) ainsi que l'utilisateur préférentiel par poste. Votre compte, votre(vos) site(s) de travail, la liste des payeurs, les pourcentages à appliquer sur les recettes , etc... Pour les analyses, les immobilisations, elles se font par utilisateur. Soyez attentifs à ça. Si vous êtes seul, la définition de l'utilisateur préférentiel permet de ne pas se tromper.

Utiliser le module recettes : Il y a 7 régions dans la fenêtre . La région "1" est définie par le paramétrage. La date est la date du jour, on peut modifier la date d'enregistrement. le site est celui que vous avez défini, par exemple votre cabinet médical. L'utilisateur utilisé est celui affiché, d'où l'intérêt de bien définir votre utilisateur préférentiel qui apparaîtra en premier. De même pour le compte en banque. La payeur est en général le patient, mais ce peut être une institution dont les noms et adresse devront être paramétrés. C'est important pour le module de gestion et de récupération des dus. La région "2" est la zone de choix des valeurs à enregistrer. Il faut noter qu'un acte préférentiel, lui même paramétré par vos soins, est affiché sur le bouton d'enregistrement. Le choix d'une autre valeur le remplacera ou si ajoutera. Le Thésaurus est une base de données de vos séries d'actes préférés, que vous créerez en enregistrant vos choix écrits sur le bouton d'enregistrement par le bouton "Plus Thésaurus". Le bouton ccam lance le logiciel de CCAM. Il permet le choix simple ou multiple d'actes CCAM, et a lui même son propre thésaurus. Lors de l'enregistrement des actes dans ce logiciel, la somme calculée selon la règle définie au dessous est affichée dans la fenêtre et les actes sur le bouton d'enregistrement. Plus loin j'explique comment nourrir la base CCAM à partir des fichiers NX. Plus bas, c'est la liste des actes NGAP, des forfaits complémentaires, et des indemnités kilométriques dont le prix unitaire est paramétrable. Il respecte la règle des abattements kilométriques. La région "3" permet de découper l'acte en pourcentage, par exemple 70 % pour la part AMO, 30 % pour la part AMC. On peut modifier ou ajouter des pourcentages qui seront stockés dans la base de données. La région "4" est une région de boutons de fonctions. Essayez les simplement. Le bouton thésaurus permet d'enregistrer une suite d'actes pour utilisation préférentielle. Le bouton "Recettes" permet de visualiser les recettes enregistrer pour les vérifier et les modifier. Un paragraphe traite du contrôle des recettes. Le bouton Error permet de voir les messages de debug et de warning du logiciel pour envoi éventuel au développeur en cas de bug. Le bouton "?" est une aide : en cliquant dessus la souris devient interrogative et si elle est amenée sur la région qui pose question et qu'on reclique à cet endroit là, un message d'explication apparaît. Le bouton Paramétrages ouvre le même programme de paramétrage que celui de comptatriée. La région "5" est la région du choix du nom et du prénom du patient. Si le logiciel est lancé par MedinTux après installation des boutons par le setup, l'ouverture du logiciel dans le dossier du patient inscrit directement son nom et son prénom dans cette région. En mode indépendant il faut soit choisir un nom dans la liste, soit en créer un nouveau sous la forme : nom;prénom ou nom prénom séparé d'un simple espace. La région "6" est la région du choix du mode de paiement. Y apparaît aussi un choix "bloquer ouvert" si le choix de "fermeture automatique" est fait. La région "7" est la région d'enregistrement des actes et sommes. C'est ce qui est inscrit sur le bouton qui est enregistré. On peut enregistrer plusieurs soins dans une même séance. Ceci pour une comptabilité complémentaire par séances.

Pour vérifier et modifier les recettes : on clique sur le bouton "recette". Une fenêtre apparaît où on peut filtrer les recettes sur plusieurs critères, et on voit le résultat en cliquant sur le bouton de filtrage. Pour filtrer sur le nom du patient, utiliser le signe "%" comme joker. Par exemple pour trouver les recettes de Mr Machin Pierre, écrivez %machin% . Vous onvservez aussi un bouton de gestion des dus qui permet de lancer le logiciel de "gestion des dus".

Le module Immobilisations permet d'enregistrer de gros achats, qui seront déclarés aux Impôts comme "immobilisés", et donc déduits des impôts de façon linéaire sur x années, ou de façon dégressive selon votre choix (voir avec votre comptable ou votre AGA).

Les modules d'analyse de la comptabilité. Ils sont lancés dans comptatriee : Ils dépendent de l'utilisateur choisi.

Le module Dépenses . Le module Dépenses permet d'enregistrer les dépenses professionnelles, ventilées selon la comptabilité des impôts. Ces dépenses sont enregistrables en pourcentage professionnels et privés. Seules seront comptabilisés les parts professionnelles. Par exemple, une facture de gaz dans un cabinet intégré à la maison qui utilise 1/3 en professionnel sera déclarée à 33%.

Donc : dans la zone "1" vous choisirez le type de dépense, la somme et le détail de la dépense (par exemple : salaires > 1500 euros > salaire de la secrétaire.) Dans la zone "2" vous vérifiez le pourcentage professionnel à appliquer à la dépense. Dans la zone "3" vous choisissez l'utilisateur concerné (votre login par exemple). Dans la zone "4" vous choisissez la date de valeur. Dans la zone "5" vous choisissez le mode de règlement. Dans la zone "6" vous validez si vous avez reçu la facture ou non. Si vous l'avez reçue en même temps que la dépense, il faut cocher la case "facture reçue". Si vous la recevez plus tard, vous sélectionnez la dépense, puis vous validez pour indiquer que vous avez reçu la facture. Ca vous permet de vous y retrouver lors de votre déclaration des impôts. Avec "Effacer" vous effacez une dépense sélectionnée. Puis une fois tout rempli, vous enregistrez la dépense par le bouton "7". Vous visualiserez les dépenses dans la fenêtre et vous pouvez les trier en cliquant sur les têtes de liste. Ca vous permet de les retrouver plus facilement.
